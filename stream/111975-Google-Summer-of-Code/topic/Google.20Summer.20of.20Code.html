Himanshu (Feb 24 2024 at 13:39):

First, you should learn about BRL-CAD, what it is and how it works:

  • Install BRL-CAD. You can find binary packages at https://github.com/BRL-CAD/brlcad/releases. This will install over 200 programs. Most of them are small command line tools. The most interesting for you are our old but still heavily used GUI mged and the more modern GUI archer.
  • Go through the mged tutorial https://brlcad.org/w/images/c/cf/Introduction_to_MGED.pdf. The tutorial will show you how to create geometries, assign materials, create ray-trace images, and many more.

Daniel Rossberg (Feb 26 2024 at 11:13):

Your compilation shows some very strange errors. Starting from scratch is probably the best.

  • Do an out-of-source build, i.e. don't use the source directory as your build directory. E.g., if BRL-CAD is your base directory for BRL-CAD developments,
    • Clone the brlcad sources to BRL-CAD/brlcad
    • Put the brlcad build files to BRL-CAD/build/brlcad
    • Install brlcad to BRL-CAD/install/brlcad
  • Use the CMake GUI for configuring the build files
    • Select your Visual Studio version there
    • After hitting "Configure" the first time, set CMAKE_INSTALL_PREFIX to the installation directory (something like BRL-CAD/install/brlcad)
    • Hit "Configure" again
    • Are there any errors during the configuration (if stuff is simply not found, it doesn't mean to be an error)
    • Hit "Generate"
  • Open the Visual Studio solution by double-clicking BRLCAD.sln in the build directory
  • Build the CMakePredefinedTargets->ALL_BUILD project
  • Open the output window. What is the first error there?
